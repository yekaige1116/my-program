Claude Code 会话录像自动备份工具

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

为什么需要这个？

Claude Code每次对话的完整记录（jsonl文件）只保存在本地。
一旦重装系统、换电脑、硬盘坏了，这些记录就全没了。

这个工具的作用：自动把记录备份到你指定的目录。
建议备份到有云同步的目录（比如OneDrive、坚果云、iCloud），
这样即使电脑出问题，云端还有一份。

怎么装？

1. 把这个文件夹放到任意位置
2. 打开终端，运行：python install.py
3. 按提示输入备份目录（建议填你的OneDrive或云盘下的路径）
4. 可选择配置 AI 主题提取功能（需要 API Key）
5. 看到 ✅ 部署完成 就搞定了

装完之后？

什么都不用做。以后每次用Claude Code：
· 每轮对话结束 → 后台自动备份（你无感知）
· 关窗口时 → 再自动备份一次（防漏）

一次安装，永久生效，完全无感。

备份了什么？

每个会话的完整录像，包括：
· 你的每一句话、AI的每一句回答
· AI的思考过程、工具调用的参数和返回值
· 时间戳、token用量等元数据

自动生成的索引
─────────────────────

除了原始会话文件，工具还会自动生成一个索引文件 sessions.json，
方便你快速搜索历史对话。

索引格式示例：
{
  "sessions": [
    {
      "date": "2026-03-04",
      "time": "13:50",
      "project": "java",
      "topic": "检测java环境是否已成功安装"
    }
  ]
}

每天凌晨 2:00 自动更新索引。

AI 主题提取（可选）
─────────────────────

安装时可选择配置 AI API，启用智能主题提取功能：
· 有 API Key：AI 会分析整个对话内容，生成精准的主题摘要
· 无 API Key：使用第一条消息的前 30 个字作为主题

支持的 AI 服务商：
· OpenAI (gpt-4o-mini)
· DeepSeek (deepseek-chat)
· 智谱 GLM (glm-4-flash)
· 其他 OpenAI 兼容接口

怎么找历史记录？
─────────────────────

方式一：直接问 Claude
  "帮我在 D:\我的备份目录\sessions.json 里搜索关于java的对话"

方式二：直接打开 sessions.json 搜索关键词

方式三：浏览备份目录，按项目分文件夹，每个会话是一个jsonl文件

怎么卸载？
─────────────────────

运行：python uninstall.py
会移除hooks、定时任务，但保留已备份的文件。
